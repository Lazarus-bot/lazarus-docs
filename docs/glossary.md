# Glossary

## Advanced Mode

A settings mode that exposes detailed detector and trading controls. Manual changes should save the profile as Custom.

## Audit Score

A score out of 100 that summarizes basic risk checks such as ownership, honeypot status, LP state, liquidity, and holder concentration.

## Community Takeover

A token revival attempt led by a community or new participants after the original team appears inactive, absent, or no longer driving the project.

## CTO

Short for community takeover.

## Detector

The part of Lazarus that evaluates activity and determines whether a token is showing comeback behavior.

## Guided Preset

A simplified configuration profile for users who do not want to tune every parameter. Initial presets are Starter, Balanced, and Aggressive.

## Live Mode

A controlled mode where trades can be submitted on-chain from a user wallet. Live mode should be gated, allowlisted during beta, and risk-controlled.

## Paper Mode

A simulation mode that records hypothetical trades using live quotes without submitting on-chain transactions.

## Signal

A detected token event or pattern that may indicate renewed activity.

## Strength Score

A score out of 100 that summarizes how strong a detected comeback signal appears.

## Tenant

The account boundary used to keep each user's settings, wallets, subscriptions, and trade activity separate.

## Watch-Only Wallet

A wallet address added for tracking. It cannot sign trades because Lazarus does not have a private key for it.

## Wallet Limit

The number of active wallets a user can configure based on their plan.
